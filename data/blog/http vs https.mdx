## HTTP 그게 뭔데?

HTTP(HyperText Transfer Protocol)는 간단히 말하자면 인터넷을 작동시키는 역할을 하며, 웹 서버
및 웹 브라우저 상호 간의 데이터 전송을 위한 응용계층 프로토콜이다. 웹 사이트에 액세스 하기 위해서는
프로토콜 변형이 필요한데, 이때 웹 사이트 URL이 일반적으로 "http://www..." 로 시작하며 URL에 해당하는
웹 페이지를 가져오기 위해 웹 사이트 서버에 명령을 보내 작동하게 된다.

## HTTPS 그게 뭔데?

HTTPS(HyperText Transfer Protocol Secure)는 표준 HTTP와 동일한 방식으로 작동한다. 서버와 주고받는
데이터가 암호화되기 때문에 웹사이트에 추가적인 보호 장치를 하는 샘이다.

#### 이미지 출처 "https://www.javatpoint.com/http-vs-https"

<br />두 프로토콜의 가장 큰 차이점은 SSL 인증서라고 한다. HTTPS는 쉽게 말해 HTTP 프로토콜에 보안
기능을 추가한 것이라고 생각하면 될 것 같다. 예를 들어 결제기능을 다루는 웹 사이트 같은 경우 사용자의
카드 정보같은 민감한 정보를 다루는 사이트는 더욱더 보안 기능이 중요하다고 볼 수 있다.

<br />
SSL 인증서는 사용자가 사이트에 제공하는 정보를 암호화하는데 데이터를 암호화 한다고 생각하면 될 것 같다.
이렇게 전송된 데이터는 중간에 누가 훔친다고 해도 데이터가 암호화 되어있기 때문에 해독을 할 수 없다.

## HTTPS의 다른 장점?

만약 개인 블로그처럼 전자상거래 기능이 없고 사용자들의 민감한 정보를 다루지 않는 웹 사이트같은 경우
HTTPS로 전환해야 할 필요성을 느끼지 못할 수도 있다. 하지만 HTTPS의 장점은 보안상에서만 우위에 있는게
아니다. HTTPS로 전환하게 되면 구글에서 SEO(검색 엔진 최적화)에 가산점을 준다고 한다. 아무래도 사용자들은
안전한 사이트에 방문하는게 당연히 더 좋다고 생각하기 때문일 것이다.
