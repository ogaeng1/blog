## SPA (Single Page Application)

단일 페이지 웹 어플리케이션을 말한다. 최초 로드 시 모든 정보를 한번에 불러온다. <br />
이후 필요한 데이터만 변경하여 사용한다. 웹사이트의 전체 페이지를 하나의 페이지에 담아 서버로부터
새로운 페이지를 불러오지 않고 화면을 동적으로 표현한다.

## SPA 장점은 뭘까? 🤔

1. 좋은 사용자 경험 제공
2. 비교적 쉬운 배포
3. 적은 서버 요청

## SPA 단점은 뭘까? 🤔

1. 페이지 첫 로드 시, 한번에 받아오기 때문에 초기 구동 속도가 느리다.
2. SEO(검색 엔진 최적화) 불리함

<br />

### MPA (Multi Page Application)

MPA는 SPA와 달리 전통적인 웹 어플리케이션 개발 방식이다. 페이지 이동 시 화면에 필요한 HTML을 서버에서
받아와 처음부터 다시 로딩하는 방식을 말한다. 받아야 할 데이터가 많다면 흰 화면이 오래 지속된다.

## MPA 장점은 뭘까? 🤔

1. SEO(검색 엔진 최적화) 유리

## MPA 단점은 뭘까? 🤔

1. 페이지 이동 시 깜빡임 현상
2. 프론트엔드와 백엔드의 연관이 밀접해 개발 복잡도가 증가할 수 있음
