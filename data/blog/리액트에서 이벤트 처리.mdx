사용자가 웹 브라우저에서 DOM 요소들과 상호 작용하는 것을 이벤트(event)라고 합니다. <br />
예를들어 흔하게 사용되는 onclick같은 이벤트가 있습니다. HTML에서는 다음과 같이 DOM 요소에 <br />
이벤트를 설정할 수 있습니다.

```html
...
<body>
  <button onclick="alert('클릭!!')">클릭</button>
</body>
...
```

이처럼 HTML에서는 이벤트를 실행하면 " " 사이에 있는 자바스크립트를 실행하도록 코드를 작성합니다.

## 리액트에서 이벤트

리액트의 이벤트 시스템은 HTML에서 사용하는 방법과 비슷합니다. 하지만 몇 가지 주의사항이 있습니다.

1. 이벤트 이름은 카멜 표기법으로 작성합니다.
   예를 들어 HTML의 onclick은 리액트에서는 onClick으로 작성해야 합니다.

2. 자바스크립트 코드가 아닌, 함수 형태의 값을 전달합니다.

```jsx
// HTML
<body>
  <button onclick="alert('클릭!!')">클릭</button>
</body>

//React
<button onClick={함수명}>
    클릭
</button>
```

3. DOM 요소에만 이벤트를 설정할 수 있습니다.
   컴포넌트 자체적으로 이벤트를 설정할 수 없습니다. div, button, input, form 등 DOM 요소에는 가능합니다.

```jsx
// App.jsx
import Test from "./Test";

const App = () => {
    return (
        <>
            <Test onClick={...} />
        </>
    )
}

//에러
```

## 이벤트 막기

HTML에서는 기본 동작을 막기 위해 다음과 같이 코드를 작성할 수 있습니다.

```html
<form onsubmit="console.log('이벤트 발생'); return false">
  <button type="submit">클릭</button>
</form>
```

리액트에서는 false를 반환해도 기본 동작을 막을 수 없습니다. 따라서 기본 동작을 막고 싶다면 <br />
반드시 preventDefault를 호출해야 합니다.

```jsx
...
const onEventHandler = (event) => {
    event.preventDefault();
    console.log('이벤트 발생');
}

return (
    <form onSubmit={onEventHandler}>
        <button type="submit">클릭</button>
    </form>
)
```
