리액트로 개발을 하다 보면 CRA(create-react-app)를 사용하게 된다. CRA는 개발 환경부터 빌드까지 모든 것이 다
갖추어져 있다. 하지만 프로젝트를 온전히 커스터마이징 하고 싶다면 CRA를 사용하지 않고 환경 설정부터 빌드 세팅까지
전부 직접 해줘야 한다. 그 중 모듈 번들러와 트랜스파일러에 대해 알아보자

## 웹팩(Webpack)이 뭘까? 🤔

![](/static/images/webpack.png)

웹팩은 오픈 소스 자바스크립트 모듈 번들러다. 여러 개의 파일을 하나의 파일로 모아주는 역할을 한다. <br />
그러면 웹팩을 왜 사용하는 걸까? <br />
이전에 모듈 번들러가 없을 때 HTML에 모든 스크립트 파일을 불러왔었다.

```html
<html>
  ...
  <body>
    <script src="index.js" />
    <script src="app.js" />
    ...
  </body>
</html>
```

그러다 ES5 부터 모듈 시스템이 등장했다고 한다. 모듈을 내보내고 모듈을 불러와서 사용이 가능하다보니 <br />
몇 가지 장점들을 얻을 수 있었다.

1. 모듈이 독립적인 실행 영역을 갖게 된다.
2. script 태그로 파일을 가져오는게 아닌 필요 함수나 변수를 가져올 수 있다.

그리고 ES6 부터 import, export 구문이 나오면서 원하는 변수나 함수를 가져오기가 쉬워졌다. 그리고 하나의 모듈만
불러오면 그 파일에서 import 하고 있는 다른 모듈들은 자동적으로 불러와지게 된다.

```html
<html>
  ...
  <body>
    <script src="index.js" />
  </body>
</html>
```

하지만 위와 같은 것들은 모든 브라우저에서 지원을 하지 않기 때문에 모든 파일들을 하나의 모듈로 모아줄 소프트웨어가
필요했고, 그래서 나온게 웹팩이라고 한다.

## 바벨(Babel)이 뭘까? 🤔

![](/static/images/babel.png)

바벨은 자바스크립트 트랜스 컴파일러다. 트랜스 컴파일러는 자바스크립트를 자바스크립트로 변환 시키는데 <br />
같은 언어로 변환을 하는 이유는 최신 버전의 자바스크립트를 구 버전의 자바스크립트로 변환하기 위해서라고 한다.

```
// Babel Input: ES2015 arrow function
[1, 2, 3].map((n) => n + 1)

// Babel Output: ES5 equivalent
[1, 2, 3].map(function (n) {
  return n + 1
})
```

ES6 이후 버전은 개발자들에게 편의성을 가져다주었지만 몇몇 브라우저들은 해당 문법을 지원하지 않은 브라우저 호환성
문제가 있었다. 그래서 최신 문법을 사용하는 많은 프로젝트를 대부분의 브라우저에 호환시키기 위해 바벨을 이용해
호환이 되는 자바스크립트 버전으로 낮춘 다음 사용하는 것이라고 한다.
